## Day 3 — HTTPS（2026-04-24）

### 改动摘要
- 新增 `frontend/nginx-ssl.conf`：
  - 80 server：ACME challenge + `/healthz`（容器健康检查用）+ `301 → https`
  - 443 server：
    - `listen 443 ssl` + `http2 on` （nginx 1.25+ 语法）
    - TLS 1.2/1.3，modern ciphers，`ssl_prefer_server_ciphers off`（客户端首选）
    - OCSP stapling + `resolver 1.1.1.1 8.8.8.8`
    - 继承原 6 项安全头 + HSTS `max-age=31536000`
    - SPA fallback + `/api/` 反向代理 + SSE 优化（`proxy_buffering off`）
  - 域名字符串暂留占位 `your-domain.com`，DEPLOYMENT.md 用 sed 替换
- 新增 `docker-compose.prod.yml`（独立，非 override）：
  - backend 不暴露端口，只在 `codingbot` bridge 网络中可达
  - frontend 暴露 80/443，挂载 `./frontend/nginx-ssl.conf:/etc/nginx/conf.d/default.conf:ro`
  - 挂载 host `/etc/letsencrypt:ro` 和 `/var/www/certbot:ro`
  - backend 挂载 `/var/log/codingbot` 作为 LOG_DIR
  - 使用 `backend/.env.production`（prod-only env）
- 新增 `docs/DEPLOYMENT.md`：12 节部署手册
  - 架构图、前置条件、.env 生成、首次 certbot standalone 签发
  - `docker compose up` 启动流程、webroot 续期 cron、日志查看、备份、滚动发布、回滚、故障排查
  - 首次上线检查清单（10 项）
